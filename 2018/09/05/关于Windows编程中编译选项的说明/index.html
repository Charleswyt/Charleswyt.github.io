关于Windows编程中编译选项的说明

2018-09-05
#C/C++, windows

运行库的简要介绍

使用Visual Studio (VS)进行Windos下的C/C++开发时，会存在运行库(C/C++ -> 代码生成-> 运行库)的选择，分别为：多线程(/MT)，多线程调试(/MTd)，多线程DLL(/MD)，多线程调试DLL(/MDd)。

编译选项 静态链接的lib 说明
/MD MSVCRT.lib 多线程、Release、DLL版本的运行时库
/MT LIBCMT.lib 多线程、Release版本的运行时库
/MDd MSVCRTD.lib 多线程、Debug、DLL版本的运行时库
/MTd LIBCMTD.lib 多线程、Debug版本的运行时库
  • /MT：Multithread Static Version，表示多线程静态的版本，编译器将LIBCMT.lib安置在OBJ文件中，使链接器使用LIBCMT.lib处理外部符号
  • /MD：Multithread and DLL Specific Version，表示多线程DLL版本，编译器将MSVCRT.lib安置在OBJ文件中，其连接到DLL的方式是静态链接，实际工作的库为MSVCR80.DLL

综上所述：

  • 静态运行时库：LIBCMT.lib
  • 动态运行时库：MSVCRT.lib + MSVCR80.DLL

运行库的选择

  • 选择/MD
    • 程序不需要静态链接运行时的库，可以减小软件的大小
    • 所有的模块都采用/MD，使用的是同一个堆，不存在A堆申请，B堆释放的问题
  • 选择/MT
    • 有些系统可能没有程序所需要版本的运行库时，程序必须把运行时库静态链接上

      注：多个模块，必须使用相同类型的运行时库，不可以混合使用
