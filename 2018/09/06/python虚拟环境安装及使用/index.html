python虚拟环境安装及使用

2018-09-06
#python

虚拟环境概述

    虚拟环境是一个将不同项目所需求的依赖分别放在独立的地方的一个工具，它给这些工程创建虚拟的python环境。它解决了“项目X依赖于版本1.x，而项目Y需要项目4.x”的两难问题，而且使你的全局site-packages目录保持干净和可管理。

virtualenv的安装及使用

  1. 使用pip完成virtualenv的安装
    pip install virtualenv
  2. 为一个工程创建一个虚拟环境
    cd “path of your project” # cd至工程目录
    # 使用默认python版本完成虚拟环境创建
    virtualenv venv # 创建虚拟环境(vnev为虚拟环境名，可随意修改)
    # 使用指定的python版本完成虚拟环境创建
    mkvirtualenv --python=“path of python” # python路径中不能包含空格
    cd “path of activate” # cd至activate所在目录(venv/Scripts)
    activate # 激活虚拟环境
    “run…” # 完成各项操作
    deactivate # 退出虚拟环境
  3. requirements.txt使用
    pip freeze > requirements.txt # 将pip包保存至requirments.txt以便后续使用
    pip install -r requirements.txt # 安装所需pip包

virtualenvwrapper-win的安装及使用

    virtualenvwrapper-win提供windows了一系列命令使得和虚拟环境工作变得愉快许多，将所有的虚拟环境都放在一个路径下。

  1. 使用pip完成virtualenvwrapper-win的安装
    pip install virtualenvwrapper-win
  2. 完成虚拟环境创建
    # 使用默认python版本完成虚拟环境创建
    mkvirtualenv venv # 环境目录路径为系统默认路径, C:\Users\Administrator\Envs\venv, 创建后自动激活虚拟环境
    # 使用指定的python版本完成虚拟环境创建
    mkvirtualenv --python=“path of python” # python路径中不能包含空格
    workon # 查看Envs目录下所有虚拟环境变量
    “run…” # 完成各项操作
    deactivate # 退出当前虚拟环境

使用Anaconda完成虚拟环境的安装及使用

  1. 完成Anaconda的安装与环境配置
  2. 打开命令行完成以下操作
    conda create -n venv python=3 # vnev为虚拟环境名，python=3选定python版本
    activate venv # 激活虚拟环境
    “run…” # 完成各项操作
    deactivate venv # 退出虚拟环境
    conda remove -n venv --all # 删除虚拟环境
