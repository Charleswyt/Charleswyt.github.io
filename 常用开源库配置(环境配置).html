配置说明

本机环境配置为win10 + vs2015 + Release + x64,其他用户需根据个人情况自行调整
配置过程中主要关注各开源库中include及libs的路径
注意：环境配置需注意配置(Debug/Release)和平台(x86/x64的选择)
dll库的环境配置：

  1. 将dll拷贝至当前exe所在目录
  2. 调试->环境，添加PATH=***,***为dll文件所在路径，非dll文件的路径，且路径不包含空格

armadillo(矩阵处理)

  1. 下载armadillo并解压至指定目录
  2. 运行Visual Studio,
    • 将***\include添加至VC++的包含目录(***为armadillo根目录)
    • 将***examples\lib_win64添加至VC++的库目录
    • 在链接器->输入->附加依赖项添加”lapack_win64_MT.lib”和”blas_win64_MT.lib”
  3. 新建一个cpp文件，输入
    #include \<armadillo>
    using namespace arma;
    若不报错则配置完成

opencv + vs2015(图像及矩阵处理)

  1. 下载所需版本的opencv库并解压至指定目录
  2. 运行Visual Studio 2015,
    • 将***\build\include, ***\build\include\opencv, ***\build\include\opencv2添加至VC++的包含目录(***为opencv根目录)
    • 将***\build\x64\vc14\lib添加至VC++的库目录(opencv版本需和vs版本对应， vs2015对应vc14)
    • 链接器->输入->附加依赖项添加，将***\build\x64\vc14\lib存在所有lib文件名复制进窗口，不同的版本略有不同，以下以opencv3.4为例
      • 对于Debug配置，添加”opencv_world342d.lib”(342表示opencv版本，根据版本自行修改)
      • 对于Release配置，添加”opencv_world342.lib”
  3. 配置opencv_world342.dll的运行环境
  4. 运行demo检查配置是否完成

jsoncpp(json读写)

  1. git clone https://github.com/open-source-parsers/jsoncpp.git
  2. cd至***\makefiles\vs71，打开jsoncpp.sln，并完成编译(***为jsoncpp跟目录)
  3. 将***\include添加至VC++的包含目录，将生成的lib文件所在路径添加至VC++库目录
  4. 链接器->输入->附加依赖项添加json_vc71_libmt(d).lib
  5. 运行demo检查配置是否完成

boost(算法)

  1. 下载所需版本的boost库并解压至指定目录
  2. 运行boost根目录下的bootstrap.bat脚本，得到bjam.exe
  3. 运行bjam.exe完成boost库的编译
  4. 运行Visual Studio，
    • 将boost根目录添加至VC++的包含目录
    • 将***\stage\lib添加至VC++的库目录(***为boost根目录)
  5. 运行demo检查配置是否完成

zlib + vs2015(压缩)

  1. 下载所需版本的zlib库并解压至指定目录
  2. 若进行32位编译：进入***\contrib\masmx86目录，运行bld_ml32.bat脚本
    若进行64位编译：进入***\contrib\masmx64目录，运行bld_ml64.bat脚本(***为zlib根目录)
    如果双击运行脚本未生成新文件，找到ml.exe(x86)或ml64.exe(x64)，复制至当前路径，双击bat脚本即可完成编译
    运行完成删除ml(64).exe文件即可(可选)
  3. 进入***\contrib\vstudio\vc14目录，打开zlibvc.sln并完成编译(平台类型(x86/x64)需和步骤2一致)
  4. 进入***\contrib\vstudio\vc14\x86\ZlibDllRelease或***\contrib\vstudio\vc14\x64\ZlibDllRelease添加至VC++的库目录
  5. 在链接器->输入->附加依赖项添加”zlibwapi.lib”
  6. 完成zlibwapi.dll环境配置
  7. 运行demo检查配置是否完成

cryptopp(加密)

  1. 下载所需版本的cryptopp库并解压至指定目录
  2. 在根目录下找到cryptest.sln，双击开启，将cryptlib设为启动项并根据项目配置及平台要求完成配置
  3. 将cryptopp根目录添加至VC++的包含目录，将编译成功的lib所在路径添加至VC++库目录
  4. 在链接器->输入->附加依赖项添加”cryptlib.lib”
  5. 运行demo检查配置是否完成

ffmpeg(音视频编解码)

ffmpeg介绍：ffmpeg有Static、Shared以及Dev三个版本

  • Static: 包含3个应用程序：ffmpeg.exe , ffplay.exe , ffprobe.exe，体积都很大，相关的DLL已经被编译到exe里面去了
  • Shared: 除了ffmpeg.exe , ffplay.exe , ffprobe.exe之外还有一些DLL，exe体积很小，在运行时到相应的DLL中调用功能
  • Dev: 开发者（developer）版本，里面包含了库文件xxx.lib以及头文件xxx.h，这个版本不含exe文件
  1. 下载ffmpeg的dev和shared版本(注意平台是x86还是x64)
  2. 将dev路径下的include添加至VC++包含目录，将dev路径下的lib添加至VC++库目录
  3. 完成shared\bin路径下所有dll文件的环境配置
  4. 运行demo检查配置是否完成

lame(MP3编解码)

  1. 下载所需版本的lame库并解压至指定目录
  2. 在***\vc_solution目录下找到vc9_lame.sln，双击开启，卸载app mp3rtp和app mp3x项目，并根据项目配置及平台要求完成配置(***为lame根目录)
  3. 编译完成后将***\include添加至VC++的包含目录，将生成的lib所在路径添加至VC++库目录
  4. 在链接器->输入->附加依赖项添加”libmp3lame.lib”
  5. 打开cmd，输入dumpbin \exports libmp3lame.lib查看API(或是在vc9_lame.sln中打开libmp3lame查看API)，或根据需求修改API
  6. 完成libmp3lame.dll环境配置
  7. 运行demo检查配置是否完成

SVM(SVM分类器)

  1. 下载SVM库并解压至指定目录
  2. 将svm.h和svm.cpp添加至项目中即可

glog(日志)

  1. 打开cmd，cd至指定目录，输入git clone https://github.com/google/glog.git
  2. 利用cmake完成glog编译，生成sln文件
  3. 打开glog.sln文件，完成相关配置(Debug/Releas, x86/x64, MT/MD)后编译即可
  4. 将***\src\windows添加至VC++的包含目录，将生成的glog.lib所在路径添加至VC++的库目录
  5. 完成glog.dll的环境配置
  6. 运行demo检查配置是否完成
